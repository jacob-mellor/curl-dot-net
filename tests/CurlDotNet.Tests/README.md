# CurlDotNet.Tests

This xUnit project drives the test strategy for CurlDotNet.

## Contents

- `CommandParserTests.cs` – canonical coverage for every supported curl flag and shell syntax nuance.
- `CommandParserSyntheticTests.cs` – synthetic fuzz/smoke tests that generate thousands of combinations.
- `CommandLineComparisonTests.cs` – runs `/usr/bin/curl` (or the platform equivalent) and ensures CurlDotNet output matches.
- `HttpHandlerTests.cs`, `IntegrationTests.cs`, `HttpbinIntegrationTests.cs` – protocol-level verification.
- `CurlUnit1300Tests.cs` – translations of historical libcurl unit scenarios.
- `SyntheticTests.cs` – edge cases that previously caused regressions.
- `TestResults/` – coverage and run artifacts (generated by CI).

## Conventions

- Prefer `Theory` + `InlineData` to keep coverage dense.
- Mirror curl's error codes with matching exception asserts.
- Always document intent in XML `<summary>` comments so DocFX surfaces the testing guidance.

## Commands

```bash
dotnet test tests/CurlDotNet.Tests/CurlDotNet.Tests.csproj --configuration Release
```

```bash
dotnet test tests/CurlDotNet.Tests/CurlDotNet.Tests.csproj --configuration Release /p:CollectCoverage=true
```
